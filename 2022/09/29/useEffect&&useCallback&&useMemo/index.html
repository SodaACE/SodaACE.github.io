useEffect && useCallback && useMemo

发表于 2022-09-29 分类于 React

useEffect

作用

​ 在该Effect Hook中，我们可以完成类似于class组件中生命周期的功能。类似于更新DOM、发送网络请求这些操作，都是属于React中更新DOM的副作用，而操作这些副作用的函数，我们通常将其放入Effect Hook中。

案例

​ 需求：我们定义一个加法计数器，当点击按钮时，计数值加1，同时浏览器的title值会随着计数值的改变而改变。首先我们看不使用hook的版本

​ 我们需要在componentDidMount以及componentDidUpdate两个生命周期中操作我们的DOM才能完成需求，这看起来显然比较复杂，如果我们在多个页面中都需要使用类似的需求，我们都要在其组件的两个生命周期中编写相同的逻辑，这显然让代码看起来十分不优雅。

​ 在class组件中实现该需求看起来略显复杂，而函数式组件中的代码实现相同的需求看起来比较简单合理。

​ 在函数组件中，我们只需要一句话便可以完成上述需求，这看起来合理，但是该操作属于函数的副作用，在编写React代码中，我们应该尽量规避编类似的代码。

​ 结合上述两个案例，再来讲解Effect Hook会方便理解很多。

基本使用

​ 我们可以通过Effect Hook来重构我们上述的代码

​ useEffect的第一个参数要求我们传入一个函数，同时它也可以返回一个函数，而返回的函数，则是在该组件被销毁的时候执行，比如我们在useEffect中监听Redux数据的变化，则我们需要在组件销毁的时候取消监听。

性能优化

​ 比如我们在useEffect中发送网络请求，同时组件中有一个计数器，当我们点击按钮增加计数值的时候，由于render会重新执行，所以整个组件又重新被渲染，然后执行组件中的代码，这样网络请求会再次发送造成性能浪费。

​ 而useEffect第二个参数就是解决类似的问题，它接受的是一个数组，里面的参数代表第一个参数函数中所依赖的项，只有当依赖项发生改变的时候，该函数才会重新执行。

​ 我们在useEffect的第二个参数写上一个空数组，代表第一个函数不依赖任何项，则只会在组件被创建的时候执行一次。

​ 在上面的例子中，我们在useEffect中第二个参数传入了count，只有count发生改变的时候，内部函数才会重新执行，所以当我们点击修改信息的按钮时，并不会执行useEffect中的函数。

useCallback

作用

​ useCallBack会返回一个函数的记忆值，在依赖不变的时候，多次定义某个函数，返回的值是相同的。通常我们使用这个hook是为了防止子组件多次渲染。

案例

​ 当我们使用普通函数版本的increment，当我们修改message的时候，Home组件会被重新渲染，这时候因为修改message的时候，组件被重新渲染，increment被重新定义，而increment作为props传给home组件，当props中的属性发生改变时, 组件本身才会被重新渲染，所以home会重新被渲染，这显然十分浪费性能。

​ 但如果我们用useCallBack给increment进行包裹，并且第二个参数传入count，则只有count改变的时候，increment才会被重新定义，所以此时再修改message时，home组件不会被重新渲染。

useMemo

作用

​ useMemo的作用和useCallback大同小异，其作用有两个，第一是为了缓存值，第二是防止子组件多次渲染。而useMemo传入的函数是需要返回的是一个函数。在官网中，也有一句话对两个的区别进行了阐述。

useCallback(fn, deps)相当于useMemo(() => fn, deps)

案例

​ 当我们将calNumTotal放入useMemo中，则我们修改计数器的值的时候，不会让result重新计算渲染，同时子组件HelloWorld也不会重新渲染（因为info对象被useMemo包裹，如果我们传入的只是普通的info对象，那么由于组件重新渲染导致info被重定义，父组件传入的props改变导致HelloWorld重新渲染）。
