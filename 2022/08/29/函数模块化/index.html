函数模块化

发表于 2022-08-29 更新于 2022-08-30 分类于 JavaScript

什么是模块化

​ 首先来看维基百科对模块化的定义：

​ 模块化编程（英语：modular programming），是强调将计算机程序的功能分离成独立的、可相互改变的模块（module）的软件设计软件设计)技术，它使得每个模块都包含着执行预期功能的一个唯一方面（aspect）所必需的所有东西。

​ 通俗易懂的来讲，我们所说的模块化开发，就是将一个程序划分成一个个小的结构，按照这个结构划分去开发程序就是所谓的模块化开发。

​ 在每个结构结构中，我们可以编写对应的逻辑代码，而且这些逻辑代码在这个结构中有独有的作用域， 在这个结构可以将自己希望暴露的变量、函数、对象等导出给其结构使用；也可以通过某种方式，导入另外结构中的变量、函数、对象等。

模块化的历史背景

​ 摘自MDN：

​ JavaScript 程序本来很小——在早期，它们大多被用来执行独立的脚本任务，在你的 web 页面需要的地方提供一定交互，所以一般不需要多大的脚本。过了几年，我们现在有了运行大量 JavaScript 脚本的复杂程序，还有一些被用在其他环境（例如 Node.js）。

​ 因此，近年来，有必要开始考虑提供一种将 JavaScript 程序拆分为可按需导入的单独模块的机制。Node.js 已经提供这个能力很长时间了，还有很多的 JavaScript 库和框架已经开始了模块的使用（例如，CommonJS 和基于 AMD 的其他模块系统 如 RequireJS， 以及最新的 Webpack 和 Babel）。

​ 所以由此我们可以看出，无论你多么喜欢JavaScript这门语言，没有模块化是早期存在的极大痛点之一，虽然社区中给出了很多方案，但是JavaScript本身，直到ES6（2015）才推出了自己的模块化方案。而在此之前，为了让JavaScript支持模块化，涌现出了很多不同的模块化规范：AMD、CMD、CommonJS等

早期没有模块化的问题及解决方案

​ 没有模块化最大的问题之一就是命名冲突。

​ 假如甲乙在编写同一个程序，然后他们在各自实现对应的子功能的js文件里使用相同的变量名，最后在合并狼个人的代码时就会出现命名冲突的问题。

​ 在早期的解决方案中，我们可以用**IIFE(立即执行函数)**来解决，将逻辑代码用立即执行函数包裹成一个模块，将这个模块进行导出

​ 但是该方案看起来显然不优雅，首先使用立即执行函数包裹代码会显得代码看起来混乱，同时没有合适的规则情况下，模块的命名也会造成混乱。

CommonJS规范

CommonJS与Node的关系

​ CommonJS这个规范在Node中是非常具有代表性的一个体现。在Node中，每一个js文件都是一个单独的模块。在这个模块中，我们使用module.exports、exports以及require三个核心变量进行开发。

​ 其中module.exports、exports用于导出模块中的内容；require用于导入其他的模块（第三方模块也可）。

AMD规范

​ AMD是Asynchronous Module Definition（异步模块定义）的缩写，它采用的是异步加载模块，而实现AMD规范我们常用的库是require.js。

​ 在main.js中配置需要导入的模块，然后在各个模块中，通过define这个函数来定义对应的模块

​ 在各个模块中，我们可以通过require来导入模块，即可获得模块中导出的内容

CMD规范

​ CMD 是Common Module Definition（通用模块定义）的缩写； 它也采用了异步加载模块，但是它将CommonJS的优点吸收了过来。这里实现CMD规范库是sea.js。

ES Module规范

初识ES Module

​ ES Module是ES2015中推出的官方的规范的模块化方式，在ES Module中采用export和import两个关键字进行导入导出，同时使用ES Module时默认是使用严格模式的。

​

export和import

​ export导出方式可以在语句前加上export，也可以将需要导出的标识符放入export的{}中

​ 而import导入就是通过export导出的标识符对应的导入，可以通过*****来表示export导出的所有变量

default用法

​ default导出的内容，我们通过import导入的时候，可以不需要{}，而且可以自己来进行对导入的内容命名，但是通过export default导出时，导出的内容只能有一个。
